Archive & Rollforward

Run once per year after the final clinic day. Complete each phase in order.

Phase 1 — Archive Current Data

Creates a copy of the entire CATBUS spreadsheet (all tabs) in the same Drive folder, named CATBUS Archive . This preserves all data before any clearing takes place.

Open Archive Spreadsheet ↗

Phase 2 — Set New Clinic Dates

Enter the four clinic dates for the coming year. These will override the hardcoded dates in Config.gs (appointment emails, schedule pages, eligibility checker) once saved.

Phase 2.5 — Archive Volunteer Stats

Reads the current Consolidated Volunteer List, SignOut sheet, and Tax Return Tracker to compute per-volunteer hours and returns for . Upserts each volunteer into the permanent Volunteer Alumni sheet (created automatically if it doesn't exist). This must run before Phase 3 clears the operational sheets.

Phase 3 — Clear Operational Data

Clears all data rows (header kept) from the year-specific operational sheets listed below. UFILE Keys and the Distribution Log are not touched.

    Manual tasks remaining:
    • Add the new year's UFILE product keys to the UFILE Keys sheet (Year column = next year, Key column = code string, Times Used = 0).
    • Verify the Volunteer Alumni sheet has the expected rows — one per volunteer, with correct returns and hours for .

    Confirm Rollforward

    This will permanently delete all data rows from:

    Client Intake • Client Assignment • Help Requests • Review Requests • Tax Return Tracker • Volunteer List • SignOut • Schedule Availability • Shift Schedule • Messages • Appointment Bookings

    UFILE Keys and the Distribution Log will not be cleared.

    This cannot be undone. Only proceed after confirming the archive is complete.
