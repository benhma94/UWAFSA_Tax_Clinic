# CATBUS Setup Guide

Step-by-step instructions for deploying a fresh CATBUS instance. The setup wizard (`setup/setup.js`) handles most of this automatically — read through once so you know what it does and what to configure afterwards.

**Estimated time: 15–30 minutes**

---

## Prerequisites

Before running the wizard, have the following ready:

| Requirement | Notes |
|---|---|
| Google account | Must own the Apps Script project — Google Workspace or Gmail |
| Google Apps Script API enabled | `script.google.com/home/usersettings` → turn ON "Google Apps Script API" |
| Node.js ≥ 16 | [nodejs.org](https://nodejs.org/) — check with `node --version` |
| clasp | Auto-installed by the wizard, or manual: `npm install -g @google/clasp` |
| Blank Google Spreadsheet | This becomes the CATBUS database |
| (Optional) Resume Drive folder | A Google Drive folder for volunteer resume uploads |
| (Optional) Quiz Drive folder | A Google Drive folder for quiz file uploads |
| (Optional) Appointment booking Google Form | For complex-case clients to book appointments |

---

## Step 1 — Initialize the Spreadsheet

Run this **before** the setup wizard so you have a Spreadsheet ID ready.

1. Create a new blank Google Spreadsheet.
2. Open **Extensions → Apps Script**.
3. Delete any existing code. Paste the full contents of [`setup/create-sheets.gs`](setup/create-sheets.gs).
4. Click **Run (▶)** with `createCatbusSheets` selected.
5. Authorize when prompted (you'll see a Google sign-in dialog).
6. A confirmation dialog lists the 15 sheets created.
7. Close the Apps Script editor. Back in the spreadsheet, copy the **Spreadsheet ID** from the URL:
   ```
   docs.google.com/spreadsheets/d/  THIS_PART  /edit
   ```

> The sheet names and column order are hardcoded in `Config.gs`. Do not rename them.

---

## Step 2 — Run the Setup Wizard

```bash
node setup/setup.js          # Mac / Linux / Windows (Git Bash)
setup\run-setup.bat          # Windows (double-click in Explorer)
```

The wizard walks through 8 steps and collects the following values:

| Prompt | Required? | Description |
|---|---|---|
| Spreadsheet ID | Yes | From Step 1 |
| Consolidated Volunteers Sheet ID | Optional | Separate volunteer-list spreadsheet; defaults to main Spreadsheet ID |
| Resume Folder ID | Optional | Google Drive folder ID for volunteer resume uploads |
| Quiz Folder ID | Optional | Google Drive folder ID for quiz file uploads |
| Clinic Email | Yes | Gmail or Workspace account that owns the Apps Script project |
| Clinic Website URL | Yes | Appears in email footers sent to clients |
| Booking Form URL | Optional | Google Form for complex-case appointment booking |
| Admin Password | Yes | Protects admin, alerts, messaging, and sign-in/out pages |

**What the wizard creates automatically:**

| File | Contents |
|---|---|
| `catbus_scripts/_Secrets.gs` | All secrets above (gitignored — never committed) |
| `catbus_scripts/.clasp.json` | Script ID pointing to your GAS project (gitignored) |
| `webpage/config.js` | Deployment URL for public web pages (gitignored) |

It then pushes the code, deploys the web app, and prints the Web App URL, Script ID, and Deployment ID. **Save all three.**

---

## Step 3 — Configure Clinic Settings

Edit [`catbus_scripts/Config.gs`](catbus_scripts/Config.gs) with your clinic-specific values. Key sections:

| Section | What to set |
|---|---|
| `SCHEDULE_CONFIG` | Clinic dates, day labels, shift times (Morning/Afternoon/Evening) |
| `INCOME_LIMITS` | Annual income thresholds for eligibility |
| `ELIGIBILITY_CONFIG` | Eligibility rules shown on the screening page |
| `APPOINTMENT_CONFIG` | Appointment booking behaviour, client ID prefix |
| `SIGN_IN_OUT` | Number of stations, station names, exception stations |
| `MESSAGING_CONFIG` | Message retention period (default: 7 days) |

Push changes after editing:

```bash
cd catbus_scripts
clasp push
clasp deploy --deploymentId YOUR_DEPLOYMENT_ID --description "Config update"
```

Or use the `/push` and `/deploy` skills if working in Claude Code.

---

## Step 4 — Sync Income Limits to HTML

If you changed any income limits in `Config.gs → INCOME_LIMITS`, you must also update **both** HTML files that mirror those values:

- `catbus_scripts/appointment_screening.html` — served by the web app
- `webpage/appointment_screening.html` — served by your public web server

Run `/check-config` in Claude Code to verify these are in sync.

---

## Step 5 — Upload Webpage Assets

The `webpage/` directory contains the public-facing website (eligibility screening, FAQ, etc.). Upload the entire folder to your web server, including `webpage/config.js` which was generated by the wizard.

**Important:** Re-upload `webpage/config.js` every time you redeploy — it contains the live Web App URL.

---

## Step 6 — Post-Setup Data Entry

Complete these in the Google Spreadsheet before the first clinic day:

| Task | Where |
|---|---|
| Add UFILE product codes | "UFILE Keys" sheet — one row per code: `Year`, `Key`, `0` |
| Add volunteer tags | "Volunteer Tags" sheet — `Volunteer Name`, `Tag` |
| Verify consolidated volunteers sheet | Must have a column named `Name` |
| Share spreadsheet with clinic team | Set appropriate Editor/Viewer permissions |

---

## Step 7 — Verify the Deployment

Open the following URLs (replace `{WEBAPP_URL}` with your actual URL):

| URL | Expected result |
|---|---|
| `{WEBAPP_URL}?app=intake` | Client intake form loads |
| `{WEBAPP_URL}?app=queue` | Queue dashboard (prompts for admin password) |
| `{WEBAPP_URL}?app=admin` | Admin dashboard (prompts for admin password) |
| `{WEBAPP_URL}?app=schedule` | Schedule dashboard |
| `{WEBAPP_URL}?app=availability` | Volunteer availability form |

## Full Configuration Checklist

Use this before the first clinic day:

### Required
- [ ] Spreadsheet initialized with `create-sheets.gs`
- [ ] `_Secrets.gs` populated (wizard or manual copy from `_Secrets.example.gs`)
- [ ] `CLINIC_EMAIL` set — must be the account running the GAS project
- [ ] `ADMIN_PASSWORD` set and shared with clinic admins
- [ ] `SCHEDULE_CONFIG` updated with correct clinic dates and shift times
- [ ] `INCOME_LIMITS` updated with current-year thresholds
- [ ] `appointment_screening.html` (both copies) synced with income limits
- [ ] `webpage/config.js` uploaded to public web server
- [ ] Web app deployed and URL verified

### Optional but Recommended
- [ ] `CONSOLIDATED_VOLUNTEERS_SHEET_ID` set if using a separate volunteer list
- [ ] `RESUME_FOLDER_ID` set for volunteer resume uploads
- [ ] `QUIZ_FOLDER_ID` set for quiz file uploads
- [ ] `BOOKING_FORM_URL` set for appointment booking
- [ ] UFILE product codes added to "UFILE Keys" sheet
- [ ] Volunteer tags added to "Volunteer Tags" sheet

## Ongoing Maintenance

| Task | Command |
|---|---|
| Push code changes | `/push` skill, or `cd catbus_scripts && clasp push` |
| Deploy a new version | `/deploy` skill — **never** bare `clasp deploy` (creates a new URL) |
| Pull edits from Apps Script editor | `/sync` skill, or `cd catbus_scripts && clasp pull` |
| Verify Config.gs ↔ HTML sync | `/check-config` skill |

## Secrets Reference

All instance secrets live in `catbus_scripts/_Secrets.gs` (gitignored). See [`catbus_scripts/_Secrets.example.gs`](catbus_scripts/_Secrets.example.gs) for the full template with comments.

| Secret | Description |
|---|---|
| `SPREADSHEET_ID` | Main Google Sheets database |
| `CONSOLIDATED_VOLUNTEERS_SHEET_ID` | External volunteer list (or same as above) |
| `RESUME_FOLDER_ID` | Drive folder for resume uploads |
| `QUIZ_FOLDER_ID` | Drive folder for quiz file uploads |
| `CLINIC_EMAIL` | Outgoing email address |
| `CLINIC_WEBSITE_URL` | Public site URL (used in email footers) |
| `BOOKING_FORM_URL` | Appointment booking Google Form URL |
| `WEBAPP_URL` | Deployed web app URL (auto-filled by wizard) |
| `ADMIN_PASSWORD` | Password for admin-gated pages |

## Troubleshooting

**`clasp push` says "Could not find .clasp.json"**
Run the setup wizard, or copy `.clasp.example.json` → `.clasp.json` and add your Script ID.

**"Script ID not found" / 403 errors**
Ensure the Google account running clasp has edit access to the GAS project.

**Income limits not showing correctly on screening page**
Update both `catbus_scripts/appointment_screening.html` and `webpage/appointment_screening.html` — they are separate files.

**Deployment URL changed unexpectedly**
You ran `clasp deploy` without `--deploymentId`. Always use the `/deploy` skill or specify `--deploymentId YOUR_ID`.

See [`catbus_scripts/README-CLASP.md`](catbus_scripts/README-CLASP.md) for additional clasp troubleshooting.
